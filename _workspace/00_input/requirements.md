# B2B Tennis 복식 리그 사이트 요구사항

## 기술 스택
- Frontend: Next.js 14 App Router + TypeScript + Tailwind CSS
- Backend: Next.js API Routes (서버 액션 포함)
- DB/Auth: Supabase (PostgreSQL + Supabase Auth)
- 배포 대상: Vercel (향후)

## 핵심 기능

### 1. 선수 관리 (Players CRUD)
- 최대 20명 내외의 선수 관리
- **등록**: 이름, 닉네임(선택) 입력하여 선수 추가
- **수정**: 선수 이름/닉네임 변경
- **삭제**: 선수 삭제 (경기 기록이 있는 경우 soft delete 처리)
- 선수 목록 화면에서 한눈에 관리 가능

### 2. 경기 기록 입력
- 복식 경기 (2 vs 2): 총 4명을 선수 목록에서 선택
  - 팀A: 선수1, 선수2
  - 팀B: 선수3, 선수4
- 경기 날짜 입력 (기본값: 오늘)
- 세트 스코어 입력:
  - 최소 1세트, 최대 3세트
  - 각 세트의 게임 점수 입력 (예: 6-4, 7-5, 6-3)
  - 세트 추가/삭제 버튼
- 저장 시 경기 결과 DB 저장

### 3. 포인트/순위 조회
- **조회 기간 선택**:
  - 특정 날짜 ~ 특정 날짜 (커스텀)
  - 이번 달 / 지난 달 (월 단위 빠른 선택)
  - 올해 / 작년 (연 단위 빠른 선택)
- **순위표 표시**:
  - 순위, 선수명, 경기수, 승/무/패, 포인트, 세트득실차, 게임득실차
  - 정렬: 포인트 → 세트득실차 → 게임득실차
- **포인트 계산**:
  - 승리: 3점
  - 무승부(세트 동수): 1점
  - 패배: 0점

### 4. 경기 기록 목록/조회
- 날짜 역순으로 경기 목록 표시
- 각 경기: 날짜, 팀A vs 팀B, 세트 스코어
- 경기 삭제 가능 (실수 입력 시 수정용)

## UI/UX 요구사항
- 모바일 우선 반응형 (스마트폰으로 주로 사용)
- 하단 탭 네비게이션 (홈/경기기록/순위/선수관리)
- 테니스 테마 (녹색 계열 + 흰색)
- 한국어 UI

## 인증
- 간단한 인증: Supabase Auth (이메일/비밀번호 또는 매직링크)
- 모든 기능은 로그인 후 사용 가능
- 관리자 구분 없이 모든 회원이 경기 기록 입력 가능

## 데이터 모델 요구사항
- 시즌 개념은 없음 (날짜 범위로 필터링)
- 한 선수가 같은 게임의 양 팀에 등록될 수 없음
- 세트 스코어는 숫자로 저장 (예: home_games=6, away_games=4)
